Workflow  ·  for each chunk 1..7:   you record → save as docs/recording/chunk-NN-<name>.mp4 → tell CIPHER the path → CIPHER verifies (4 gates + chunk-specific) → PASS → next chunk  |  FAIL (reason) → re-record that chunk only.

Hard rules

  • Never record the next chunk before CIPHER PASSes the current one. A bad chunk discovered late wastes everything after it.
  • No secret on screen, ever. Relay key, Telegram bot token, Solana keypair — none may appear. Drivers load them from env off-camera. If a secret leaks into a frame, delete that clip immediately + re-record.
  • One clip per chunk (chunk-01-status.mp4, …) in docs/recording/. ffmpeg stitches them at the end.
  • System audio OFF. Voiceover is recorded separately. The phone notification sound in chunk 4 is nice-to-have, not required.

The 4 gates — what CIPHER checks on every chunk

  1. Legibility — every terminal line readable at 1×. (Else: bigger font, re-record.)
  2. No secrets — Relay key, Telegram token, Solana keypair contents never on screen. (Else: delete clip, re-record.)
  3. The beat landed — the chunk-specific checklist item happened.
  4. Clean frame — clear between chunks, no notifications, no scrollback bleed.
